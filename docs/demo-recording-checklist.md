# Demo recording checklist

## За 15 минут до записи

- [ ] `make up`; дождаться healthy PostgreSQL и API.
- [ ] `make e2e`; убедиться, что полный путь зелёный.
- [ ] Открыть `http://localhost:3000` в Chromium, zoom 100%, viewport не меньше 1440×900.
- [ ] Очистить `localStorage`/старую вкладку QADAM и отключить уведомления ОС.
- [ ] Положить `demo/contracts/qadam-risky-contract.pdf` в удобное место выбора файла.
- [ ] Открыть `docs/demo-script.md` на втором экране; включить таймер 3:00.
- [ ] Проверить микрофон, 1080p, читаемый курсор и отсутствие приватных вкладок/секретов.

## Обязательные кадры

- [ ] Landing: одна аудитория, одна задача, privacy и disclaimer.
- [ ] Upload + consent + короткое состояние обработки.
- [ ] Summary severity и finding про немедленное расторжение.
- [ ] Фрагмент договора, действие и официальная статья «Әділет».
- [ ] Grounded question про депозит.
- [ ] После основной записи, если остаётся время на jury backup: mixed RU/KZ PDF → «Какая арендная
  плата?» → `160 000`, затем «Как решить найденные проблемы?» → action mode с evidence.
- [ ] Сообщение арендодателю по выбранному пункту.
- [ ] Техническое доказательство: тесты/RAG/E2E без показа секретов.

## После записи

- [ ] Продолжительность `≤ 03:00`; начало и конец без пустых секунд.
- [ ] Текст читается в 1080p, голос без клиппинга, персональные данные отсутствуют.
- [ ] Видео воспроизводится от начала до конца после повторной загрузки/экспорта.
- [ ] Public/unlisted URL открывается в приватном окне без авторизации.
- [ ] URL внесён в pitch slide 11 и submission form.
- [ ] Финальный pitch PDF остаётся `≤ 12` страниц.

## Резерв при срыве записи

- [ ] Выполнить `make fallback-demo` и проверить `release/fallback-demo-manifest.json`.
- [ ] Просмотреть `release/QADAM_AI_fallback_demo.mp4` целиком: 48 секунд, 1920×1080, подписи
  читаются, персональные данные отсутствуют.
- [ ] При необходимости записать поверх короткую командную озвучку и снова проверить `≤03:00`.
- [ ] Не называть silent fallback финальным narrated demo, пользовательским тестом или CustDev.
- [ ] Следовать [submission runbook](submission-runbook.md) для загрузки и проверки public URL.

Не записывать реальный договор. Синтетический demo PDF и его исходный `.txt` находятся в
репозитории, поэтому поведение воспроизводимо и не раскрывает данные участника.
